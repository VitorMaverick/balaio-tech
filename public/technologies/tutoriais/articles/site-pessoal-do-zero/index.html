Tutorial: criando um site pessoal do zero

Por Maverick Dev • 3 de fevereiro de 2025

Tutoriais

Todo dev precisa de um site pessoal. Sério. É o seu cartão de visita na internet, o lugar onde você mostra quem é, o que faz e o que já construiu. E a melhor parte? Você não precisa de framework nenhum pra isso. HTML e CSS puro dão conta do recado — e de quebra você pratica os fundamentos.

Nesse tutorial, vou te guiar passo a passo na criação de um site pessoal completo. Nada de template pronto, nada de copiar e colar sem entender. Vamos construir tudo do zero, entendendo cada pedaço. No final, você vai ter um site bonito, responsivo e pronto pra colocar no ar.

O que vamos construir

Nosso site vai ter:

  • Uma seção de apresentação (hero)
  • Uma seção "Sobre mim"
  • Uma seção de projetos
  • Um rodapé com links de contato

Tudo responsivo, ou seja, vai funcionar bem tanto no computador quanto no celular. Bora lá.

Passo 1: Estrutura de arquivos

Cria uma pasta pro seu projeto. Dentro dela, vamos ter dois arquivos:

  • index.html — a estrutura do site
  • style.css — toda a estilização

Simples assim. Sem node_modules, sem build, sem configuração. Abre o VS Code, cria a pasta, e vamos começar pelo HTML.

Passo 2: O HTML semântico

HTML semântico significa usar as tags certas pro conteúdo certo. Em vez de colocar tudo dentro de <div>, usamos tags como <header>, <main>, <section> e <footer>. Isso ajuda na acessibilidade e no SEO.

Aqui vai a estrutura completa do nosso index.html:

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Maverick Dev — Portfólio</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <header class="header">
        <nav class="nav">
            <a href="#" class="logo">Maverick Dev</a>
            <ul class="nav-links">
                <li><a href="#sobre">Sobre</a></li>
                <li><a href="#projetos">Projetos</a></li>
                <li><a href="#contato">Contato</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section class="hero">
            <h1>Olá, eu sou o <span>Maverick</span></h1>
            <p>Estudante de tecnologia, desenvolvedor em formação
            e apaixonado por criar coisas pra web.</p>
            <a href="#projetos" class="btn">Ver meus projetos</a>
        </section>

        <section id="sobre" class="sobre">
            <h2>Sobre mim</h2>
            <p>Tenho 20 anos, estudo Sistemas de Informação
            e estou sempre aprendendo algo novo. Atualmente
            focado em desenvolvimento web com HTML, CSS,
            JavaScript e React.</p>
        </section>

        <section id="projetos" class="projetos">
            <h2>Projetos</h2>
            <div class="projetos-grid">
                <article class="projeto-card">
                    <h3>Blog Balaio.tech</h3>
                    <p>Blog pessoal sobre tecnologia
                    feito com HTML e CSS.</p>
                    <a href="#">Ver projeto →</a>
                </article>
                <article class="projeto-card">
                    <h3>Lista de Tarefas</h3>
                    <p>App de to-do list feito com
                    JavaScript vanilla.</p>
                    <a href="#">Ver projeto →</a>
                </article>
                <article class="projeto-card">
                    <h3>Calculadora</h3>
                    <p>Calculadora funcional com
                    interface clean.</p>
                    <a href="#">Ver projeto →</a>
                </article>
            </div>
        </section>
    </main>

    <footer id="contato" class="footer">
        <h2>Contato</h2>
        <p>Me encontre por aí:</p>
        <ul class="contato-links">
            <li><a href="#">GitHub</a></li>
            <li><a href="#">LinkedIn</a></li>
            <li><a href="#">Email</a></li>
        </ul>
    </footer>
</body>
</html>

Repara em algumas coisas importantes aqui:

  • O lang="pt-BR" diz pro navegador que o conteúdo é em português
  • A meta viewport é obrigatória pra responsividade funcionar
  • Usamos <section> pra cada bloco de conteúdo, não <div>
  • Os id nas sections permitem a navegação com âncoras
  • Cada projeto é um <article> porque é um conteúdo independente

Passo 3: Reset e estilos base

Agora vamos pro CSS. Primeiro, um reset básico pra tirar os estilos padrão do navegador e definir nossas variáveis:

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

:root {
    --cor-fundo: #0f0f0f;
    --cor-texto: #e0e0e0;
    --cor-destaque: #64ffda;
    --cor-card: #1a1a2e;
    --fonte-principal: 'Segoe UI', sans-serif;
}

body {
    font-family: var(--fonte-principal);
    background-color: var(--cor-fundo);
    color: var(--cor-texto);
    line-height: 1.6;
}

a {
    color: var(--cor-destaque);
    text-decoration: none;
}

a:hover {
    text-decoration: underline;
}

O box-sizing: border-box é essencial — faz com que padding e border sejam incluídos no tamanho total do elemento. Sem isso, seus layouts vão quebrar de formas misteriosas.

As variáveis CSS (custom properties) facilitam demais a manutenção. Quer mudar a cor de destaque do site inteiro? Muda em um lugar só.

Passo 4: Estilizando o header e navegação

.header {
    padding: 1.5rem 2rem;
    position: fixed;
    width: 100%;
    top: 0;
    background-color: rgba(15, 15, 15, 0.9);
    backdrop-filter: blur(10px);
    z-index: 100;
}

.nav {
    display: flex;
    justify-content: space-between;
    align-items: center;
    max-width: 1100px;
    margin: 0 auto;
}

.logo {
    font-size: 1.5rem;
    font-weight: 700;
    color: var(--cor-texto);
}

.nav-links {
    display: flex;
    list-style: none;
    gap: 2rem;
}

.nav-links a {
    color: var(--cor-texto);
    transition: color 0.3s;
}

.nav-links a:hover {
    color: var(--cor-destaque);
    text-decoration: none;
}

O header fixo com backdrop-filter: blur() dá aquele efeito de vidro fosco que fica muito bonito em temas escuros. O z-index: 100 garante que ele fica acima de todo o resto do conteúdo.

Passo 5: A seção hero

.hero {
    min-height: 100vh;
    display: flex;
    flex-direction: column;
    justify-content: center;
    align-items: center;
    text-align: center;
    padding: 2rem;
}

.hero h1 {
    font-size: 3rem;
    margin-bottom: 1rem;
}

.hero h1 span {
    color: var(--cor-destaque);
}

.hero p {
    font-size: 1.2rem;
    max-width: 500px;
    margin-bottom: 2rem;
    color: #aaa;
}

.btn {
    padding: 0.8rem 2rem;
    border: 2px solid var(--cor-destaque);
    border-radius: 5px;
    color: var(--cor-destaque);
    font-weight: 600;
    transition: background-color 0.3s, color 0.3s;
}

.btn:hover {
    background-color: var(--cor-destaque);
    color: var(--cor-fundo);
    text-decoration: none;
}

O min-height: 100vh faz a seção hero ocupar a tela inteira. O Flexbox centraliza tudo vertical e horizontalmente. É a combinação mais usada no desenvolvimento web moderno pra centralizar conteúdo.

Passo 6: Seções de conteúdo e grid de projetos

.sobre, .projetos {
    max-width: 1100px;
    margin: 0 auto;
    padding: 5rem 2rem;
}

.sobre h2, .projetos h2 {
    font-size: 2rem;
    margin-bottom: 1.5rem;
    color: var(--cor-destaque);
}

.projetos-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
    gap: 1.5rem;
}

.projeto-card {
    background-color: var(--cor-card);
    padding: 2rem;
    border-radius: 10px;
    border: 1px solid #2a2a3e;
    transition: transform 0.3s, border-color 0.3s;
}

.projeto-card:hover {
    transform: translateY(-5px);
    border-color: var(--cor-destaque);
}

.projeto-card h3 {
    margin-bottom: 0.8rem;
    font-size: 1.3rem;
}

.projeto-card p {
    color: #aaa;
    margin-bottom: 1rem;
}

Aqui tá a mágica da responsividade sem media query: grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)). Essa linha sozinha faz os cards se reorganizarem automaticamente conforme o tamanho da tela. Em telas grandes ficam 3 colunas, em tablets 2, no celular 1. Tudo automático.

Passo 7: Footer e responsividade

.footer {
    text-align: center;
    padding: 4rem 2rem;
    border-top: 1px solid #2a2a3e;
}

.footer h2 {
    color: var(--cor-destaque);
    margin-bottom: 1rem;
}

.contato-links {
    list-style: none;
    display: flex;
    justify-content: center;
    gap: 2rem;
    margin-top: 1rem;
}

/* Responsividade */
@media (max-width: 768px) {
    .hero h1 {
        font-size: 2rem;
    }

    .hero p {
        font-size: 1rem;
    }

    .nav-links {
        gap: 1rem;
    }

    .contato-links {
        flex-direction: column;
        gap: 0.8rem;
    }
}

A media query cuida dos ajustes finos que o grid sozinho não resolve — como tamanho de fonte e espaçamento que ficam grandes demais no celular.

Dicas extras que fazem diferença

Agora que o site tá pronto, algumas dicas pra deixar ele ainda melhor:

  • Smooth scroll: adicione html { scroll-behavior: smooth; } no CSS. Os links de âncora vão rolar suavemente em vez de pular
  • Favicon: adicione um favicon pro site ter aquele ícone na aba do navegador. Pode usar o favicon.io pra gerar um
  • Meta description: adicione <meta name="description" content="Portfólio de Maverick Dev"> no head pra melhorar o SEO
  • Fonte do Google Fonts: uma fonte boa muda completamente a cara do site. Inter, Poppins ou Fira Code são ótimas opções

Colocando no ar

Com o site pronto, você pode hospedá-lo de graça no GitHub Pages. É só criar um repositório, subir seus arquivos e ativar o Pages nas configurações. Em 5 minutos seu site tá online com uma URL tipo seuusuario.github.io.

Outra opção é a Vercel ou Netlify — só arrastar a pasta do projeto pra lá e pronto, tá no ar. Sem configuração nenhuma.

O melhor portfólio é aquele que existe. Não espere ficar perfeito pra publicar. Coloca no ar, vai melhorando com o tempo.

Conclusão

Com HTML e CSS puro, sem nenhum framework, a gente construiu um site pessoal completo, bonito e responsivo. Esse é o tipo de projeto que mostra pra um recrutador que você domina os fundamentos — e fundamentos sólidos são o que separam quem realmente entende web de quem só copia tutorial.

Pega esse código, personaliza com as suas informações, muda as cores, adiciona seus projetos reais. Faz ele seu. E o mais importante: coloca no ar. Um site publicado vale mais que dez projetos na sua máquina que ninguém nunca viu.

Se ficou com alguma dúvida ou quer mostrar o resultado, me chama lá no GitHub. Bora construir coisas juntos.
