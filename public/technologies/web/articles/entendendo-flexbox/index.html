Entendendo Flexbox de uma vez por todas

Por Maverick Dev • 1 de dezembro de 2024

Web

Por que Flexbox?

Se você já tentou colocar dois elementos lado a lado no CSS e tudo desandou, bem-vindo ao clube. Antes do Flexbox, a gente usava float, inline-block, position absolute — tudo gambiarra. Funcionava, mas era sofrido.

Flexbox (Flexible Box Layout) surgiu pra resolver exatamente isso: organizar elementos em uma direção (linha ou coluna) de forma previsível e flexível. Depois que você entende como funciona, nunca mais vai querer voltar pros métodos antigos.

Vou explicar do zero, com exemplos práticos. Ao final desse artigo, você vai saber posicionar qualquer coisa com Flexbox.

O conceito fundamental: container e items

Flexbox trabalha com dois papéis: o flex container (pai) e os flex items (filhos). Você ativa o Flexbox no elemento pai, e os filhos diretos se tornam os items flexíveis.

<div class="container">
    <div class="item">1</div>
    <div class="item">2</div>
    <div class="item">3</div>
</div>
.container {
    display: flex;
}

Só isso. Com essa única linha, os três items que antes ficavam empilhados (um embaixo do outro) agora ficam lado a lado. Mágico? Quase.

Os dois eixos: principal e cruzado

Esse é o conceito mais importante do Flexbox. Tudo gira em torno de dois eixos:

  • Eixo principal (main axis): a direção em que os items são distribuídos. Por padrão, é horizontal (da esquerda pra direita).
  • Eixo cruzado (cross axis): perpendicular ao eixo principal. Por padrão, é vertical (de cima pra baixo).

Por que isso importa? Porque as propriedades de alinhamento funcionam em relação a esses eixos. justify-content alinha no eixo principal. align-items alinha no eixo cruzado. Se você confunde os dois, o resultado não é o que espera.

flex-direction: mudando o eixo principal

Por padrão, o eixo principal é horizontal. Mas você pode mudar com flex-direction:

.container {
    display: flex;
    flex-direction: row;         /* padrão: horizontal */
}

.container {
    display: flex;
    flex-direction: column;      /* vertical: items empilhados */
}

.container {
    display: flex;
    flex-direction: row-reverse; /* horizontal invertido */
}

.container {
    display: flex;
    flex-direction: column-reverse; /* vertical invertido */
}

Quando você usa column, os eixos se invertem: o principal vira vertical e o cruzado vira horizontal. Isso pega muita gente de surpresa. Se o justify-content não está funcionando como você espera, confere a flex-direction — provavelmente os eixos estão trocados.

justify-content: alinhamento no eixo principal

Essa propriedade controla como os items se distribuem ao longo do eixo principal. Os valores mais usados:

.container {
    display: flex;
    justify-content: flex-start;    /* items no início (padrão) */
}

.container {
    display: flex;
    justify-content: flex-end;      /* items no final */
}

.container {
    display: flex;
    justify-content: center;        /* items centralizados */
}

.container {
    display: flex;
    justify-content: space-between; /* espaço igual ENTRE os items */
}

.container {
    display: flex;
    justify-content: space-around;  /* espaço igual AO REDOR dos items */
}

.container {
    display: flex;
    justify-content: space-evenly;  /* espaço IGUAL em tudo */
}

O space-between é meu favorito. Ele joga o primeiro item pro início, o último pro final, e distribui o espaço igualmente entre os do meio. Perfeito pra navbars, por exemplo: logo na esquerda, menu na direita.

.navbar {
    display: flex;
    justify-content: space-between;
    align-items: center;
    padding: 1rem 2rem;
}

/* Logo fica na esquerda, nav fica na direita, automaticamente */

align-items: alinhamento no eixo cruzado

Enquanto justify-content cuida do eixo principal, align-items cuida do eixo cruzado. Os valores:

.container {
    display: flex;
    align-items: stretch;     /* items esticam pra ocupar toda a altura (padrão) */
}

.container {
    display: flex;
    align-items: flex-start;  /* items no topo */
}

.container {
    display: flex;
    align-items: flex-end;    /* items na base */
}

.container {
    display: flex;
    align-items: center;      /* items centralizados verticalmente */
}

.container {
    display: flex;
    align-items: baseline;    /* alinha pela linha base do texto */
}

O combo mais famoso do CSS — centralizar algo vertical e horizontalmente — fica ridiculamente simples:

.container {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh; /* precisa ter altura definida */
}

Três linhas (quatro com o height). Antes do Flexbox, isso era um pesadelo. Sério, pesquisa "how to center a div" e vê quantas soluções malucas existiam antes.

flex-wrap: permitir quebra de linha

Por padrão, os flex items tentam caber todos em uma linha, mesmo que isso signifique esmagar cada um. Se você quer que eles quebrem pra próxima linha quando não couberem, usa flex-wrap:

.container {
    display: flex;
    flex-wrap: nowrap; /* padrão: tudo em uma linha */
}

.container {
    display: flex;
    flex-wrap: wrap;   /* quebra pra próxima linha se não couber */
}

.container {
    display: flex;
    flex-wrap: wrap-reverse; /* quebra pra linha de CIMA */
}

Isso é essencial pra layouts responsivos. Imagina uma grade de cards: em tela grande cabem 3 por linha, em tela média 2, e no celular 1. Com flex-wrap: wrap e uma largura mínima nos items, isso acontece naturalmente:

.cards {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}

.card {
    flex: 1 1 300px; /* cresce, encolhe, base mínima de 300px */
}

gap: espaçamento entre items

Antes a gente usava margin nos items pra criar espaço entre eles. O problema é que sobrava margin no primeiro ou último item. A propriedade gap resolve isso de forma limpa:

.container {
    display: flex;
    gap: 1rem;         /* espaço igual em todas as direções */
}

.container {
    display: flex;
    gap: 1rem 2rem;    /* 1rem vertical, 2rem horizontal */
}

O gap só adiciona espaço ENTRE os items, nunca nas bordas. Exatamente o que queremos na maioria dos casos. Funciona com Flexbox e Grid.

Propriedades dos items

Até agora falamos de propriedades do container. Mas os items também têm propriedades úteis:

.item {
    flex-grow: 1;   /* permite que o item cresça */
    flex-shrink: 1; /* permite que o item encolha */
    flex-basis: 200px; /* tamanho base antes de crescer/encolher */
}

/* Atalho: */
.item {
    flex: 1 1 200px; /* grow shrink basis */
}

/* Item que não cresce nem encolhe: */
.item-fixo {
    flex: 0 0 200px;
}

/* Item que ocupa todo espaço disponível: */
.item-expansivel {
    flex: 1;
}

E o align-self permite que um item específico se alinhe diferente dos outros:

.container {
    display: flex;
    align-items: flex-start; /* todos no topo */
}

.item-especial {
    align-self: flex-end;    /* esse vai pra base */
}

Exemplo prático: layout de página completo

Vamos juntar tudo num exemplo real — um layout de página com header, conteúdo principal e footer:

<body class="page">
    <header class="header">Header</header>
    <main class="content">Conteúdo</main>
    <footer class="footer">Footer</footer>
</body>
.page {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}

.header {
    padding: 1rem;
}

.content {
    flex: 1; /* ocupa todo espaço disponível */
    padding: 2rem;
}

.footer {
    padding: 1rem;
}

O flex: 1 no content faz ele expandir pra empurrar o footer pra baixo, independente da quantidade de conteúdo. Sem gambiarras, sem position fixed no footer. Limpo e funcional.

Dicas finais

  • Use o DevTools: No Chrome, quando você inspeciona um elemento com display: flex, aparece um badge "flex" clicável que mostra os eixos visualmente. É ótimo pra debugar.
  • Pratique com Flexbox Froggy: É um joguinho que ensina Flexbox posicionando sapos em folhas. Parece bobo, mas ajuda muito a fixar os conceitos. Pesquisa "Flexbox Froggy" no Google.
  • Não tenha medo de testar: Muda os valores, vê o que acontece. Flexbox é muito visual — o feedback é imediato.
  • Flexbox vs Grid: Flexbox é pra layout em UMA dimensão (linha OU coluna). Grid é pra DUAS dimensões (linhas E colunas). Eles se complementam, não competem.
Resumo mental: justify-content = eixo principal. align-items = eixo cruzado. Se a direction é row, principal é horizontal. Se é column, principal é vertical. É só isso.

Flexbox é daquelas coisas que parece complexa até você praticar por uma semana. Depois vira segunda natureza. Vai por mim: gasta um tempo fazendo layoutzinhos variados e em pouco tempo você não vai nem pensar antes de usar.
